## What is Habulator?

Habulator is a machine learning emulator for predicting phytoplankton biovolume in the **Great Lakes (2001–2022)**. Given seven environmental conditions, it returns:

- **Point prediction** in mg/L (Duan smearing–corrected)
- **90% prediction interval** via split conformal prediction
- **Feature-level SHAP explanation** — why the model predicted that value

Supports five phytoplankton groups: Early Diatoms, Late Diatoms, Chlorophytes, Cryptophytes, Cyanobacteria.

## Quick Start

### Prerequisites
- Node.js ≥ 18
- Python ≥ 3.10
- Your trained model files (see [Model Files](#model-files))

### Run locally

```bash
# 1. Clone
git clone https://github.com/rohitshukla01/habulator-app.git
cd habulator-app

# 2. Backend
cd api
pip install -r requirements.txt
# Place model files in api/ (see Model Files section)
uvicorn main:app --reload --port 8000

# 3. Frontend (new terminal)
cd web
npm install
npm run dev
# → http://localhost:3000
```

## Model Files

The API expects these files in the `api/` directory (generated by the training notebooks):

```
api/
├── ediat_model.json            ← XGBoost model
├── ediat_conformal_scores.json ← Nonconformity scores for 90% PI
└── ediat_webapp_config.json    ← Smearing factor + feature config
```

> **Note:** Model files are not committed to this repo (generated by HPC training runs). See [training notebooks](https://github.com/rohitshukla01/habulator-notebooks) for reproduction.

**Demo mode:** If model files are absent, the API returns realistic dummy predictions so the frontend can be tested independently.

## Inputs & Outputs

### Inputs (7 raw features)

| Feature | Description | Unit | Valid Range |
|---|---|---|---|
| `TEMP` | Surface water temperature | °C | 0 – 30 |
| `TP` | Total phosphorus | µg/L | 0.1 – 200 |
| `SI` | Silica | mg/L | 0 – 5 |
| `NO23` | Nitrate + Nitrite | mg/L | 0 – 3 |
| `Secchi_m` | Secchi depth (water clarity) | m | 0.1 – 40 |
| `STN_DEPTH_M` | Station depth | m | 1 – 400 |
| `DOY` | Day of year | — | 1 – 365 |

### Outputs

| Field | Description |
|---|---|
| `pred_mgL` | Best estimate biovolume (Duan-corrected) |
| `lower_mgL` | 90% PI lower bound |
| `upper_mgL` | 90% PI upper bound |
| `shap` | Per-feature SHAP values (log space) |

## Deployment

### Backend → Render (free tier)

1. Push this repo to GitHub
2. Go to [render.com](https://render.com) → **New Web Service**
3. Connect your GitHub repo → select `api/` as root directory
4. Build command: `pip install -r requirements.txt`
5. Start command: `uvicorn main:app --host 0.0.0.0 --port $PORT`
6. Upload model files via Render's **Environment → Secret Files**

### Frontend → Vercel (free tier)

1. Go to [vercel.com](https://vercel.com) → **Add New Project**
2. Import this repo → set **Root Directory** to `web`
3. Add environment variable:
   ```
   NEXT_PUBLIC_API_URL = https://your-api.onrender.com
   ```
4. Deploy → live in ~2 minutes

## Methods

- **Model**: XGBoost regressor, 100-trial Optuna hyperparameter optimisation (5-fold CV)
- **Target transform**: log(1+x) — handles zero-inflated, right-skewed biovolume
- **Back-transform**: Duan smearing correction for unbiased mg/L predictions
- **Uncertainty**: Split conformal prediction (90% nominal coverage, calibrated on held-out test set)
- **Explainability**: SHAP TreeExplainer (exact Shapley values for tree models)
- **Data**: USEPA GLNPO monitoring programme, Great Lakes 2001–2022, n ≈ 2,400 after QC
